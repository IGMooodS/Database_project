# Филателистическое агентство: Модель базы данных

**Дисциплина:** Базы данных  

## О проекте

Данный репозиторий содержит решение двух практических работ по курсу баз данных. В рамках проекта разработана полноценная реляционная модель базы данных для автоматизации деятельности **Филателистического агентства** — организации, занимающейся учетом и распределением филателистической продукции (марок, открыток, конвертов и т.п.) между управлениями федеральной почтовой связи (УФПС) по всей России.

Проект включает в себя проектирование схемы БД, ее реализацию на SQL, а также разработку сложных аналитических отчетов, хранимых процедур и триггеров для учета заявок и движения продукции.

## Структура репозитория

- **`P_1/`** — первая практическая работа. Проектирование структуры БД:
  - ER-диаграмма
  - SQL-скрипты создания таблиц
  - Заполнение справочников начальными данными
- **`P_2/`** — вторая практическая работа. Разработка бизнес-логики и отчетов:
  - Сложные запросы для формирования отчетов (№1-8)
  - Хранимые процедуры и функции для автоматизации
  - Триггеры для поддержки целостности данных
  - Настройка ролей и разграничение доступа
- **`variants.pdf`** — файл с вариантами заданий, выданный преподавателем.
- **`README.md`** — данный файл с описанием проекта.

> **Примечание:** Подробные SQL-скрипты и документация находятся в соответствующих папках.


## Реализованные аналитические отчеты

Для предметной области разработаны следующие отчеты (запросы), покрывающие ключевые бизнес-задачи:

1.  **Потребность в продукции:** Получить список всех УФПС, которым необходим заданный вид ФП.
2.  **Неактивные клиенты:** Получить список УФПС, которые не оставляли никакой заявки за последние три месяца.
3.  **Сравнение заказов:** Получить список УФПС, которые заказали те же ФП, что и заданное УФПС в указанный день. Вывести: УФПС – ФП – Количество в заказе.
4.  **Частые заказы:** Получить список ФП, которые заказало указанное УФПС больше двух раз.
5.  **Лимитированные заявки:** Получить список таких УФПС, которым требуется не больше заданного количества заданной ФП. Вывести: Номер заявки – УФПС – Количество – Дата заявки.
6.  **Региональный спрос за неделю:** Получить тройки УФПС – Вид ФП – Количество ФП, расположенных в заданном регионе, по заявкам последней недели. Если от одного УФПС поступило несколько заявок на ФП одного вида, выводится суммарное количество.
7.  **Популярность видов:** Получить список из двух видов ФП (самый популярный и самый непопулярный). Вывести: Вид ФП – Количество заявок на него – Дата последней заявки.
8.  **Топ продукции за месяц:** Получить список из трёх самых популярных ФП указанного вида за указанный месяц. Вывести: ФП – Процент от общего количества ФП того же вида.
